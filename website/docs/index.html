MCUlinux Documentation

Getting Started

MCUlinux is a lightweight Alpine Linux distribution for ESP32-S3 microcontrollers. It provides a familiar Linux environment with package management capabilities.

Note: MCUlinux currently supports ESP32-S3 only. Support for other ESP32 variants is planned for future releases.

Supported Devices

Device PSRAM Flash Description
r8n8 8MB 8MB Basic configuration
r8n16 8MB 16MB Extended storage
r16n16 16MB 16MB High memory configuration

Installation

Requirements

  • ESP32-S3 development board
  • USB cable for programming and serial console
  • esptool.py or web flasher

Using Web Flasher

  1. Open MCUlinux Flasher
  2. Select your device type
  3. Download or select firmware file
  4. Connect your device
  5. Click Flash

Using esptool.py

# Install esptool
pip install esptool

# Download latest release
wget https://github.com/mculinux/mculinux/releases/latest/download/r8n8.zip
unzip r8n8.zip

# Flash firmware
esptool.py --chip esp32s3 --port /dev/ttyUSB0 --baud 115200 \
    write_flash 0x0 \
    bootloader.bin \
    partitions.bin \
    firmware.bin

Configuration

Boot Configuration

MCUlinux uses ESP-IDF's menuconfig system. To modify boot settings:

# Enter menuconfig
idf.py menuconfig

# Key settings:
# - Bootloader → Boot menu timeout
# - Partition Table → Custom partition table
# - Compiler options → Optimization level

Network Configuration

# Configure WiFi
wpa_passphrase "SSID" "password" > /etc/wpa_supplicant.conf

# Start network
wpa_supplicant -B -i wlan0 -c /etc/wpa_supplicant.conf
udhcpc -i wlan0

Packages

MCUlinux uses Alpine Linux's package manager (apk) for software management.

Available Packages

Package Description
busybox Lightweight utilities
htop Process viewer
btop Resource monitor
nano Text editor
coreutils Basic UNIX utilities
bash Shell

Installing Packages

# Update package index
apk update

# Install package
apk add htop

# Install multiple packages
apk add htop btop nano

Networking

WiFi

# Scan for networks
iwlist wlan0 scan

# Connect to network
wpa_supplicant -B -i wlan0 -c /etc/wpa_supplicant.conf
udhcpc -i wlan0

# Test connection
ping -c 3 google.com

SSH

# Install OpenSSH
apk add openssh

# Generate host keys
ssh-keygen -A

# Start SSH server
sshd

# Connect from another machine
ssh root@<device-ip>

Serial Console

MCUlinux provides a serial console for direct communication.

Connecting

# Linux/macOS
screen /dev/ttyUSB0 115200

# Windows (PuTTY)
# Serial line: COMx
# Speed: 115200

Default Credentials

Service Username Password
Serial Console root (none)
SSH root (none)

OTA Updates

MCUlinux supports Over-The-Air updates via the partition table.

OTA Process

# Check current version
cat /etc/mculinux/version

# Download new firmware
wget http://server/firmware.bin

# Apply update
mculinux-ota apply firmware.bin

# Reboot
reboot
Warning: OTA updates may cause data loss. Backup important data before updating.

Troubleshooting

Common Issues

Device not booting

  • Check USB connection
  • Verify baud rate (115200)
  • Try different USB port/cable

No serial output

  • Ensure correct COM port
  • Check baud rate settings
  • Press RESET button

WiFi not connecting

  • Verify SSID and password
  • Check signal strength
  • Ensure WiFi antenna is connected
