Getting Started

MCUlinux runs a full Linux kernel on the ESP32-S3 — a $5 microcontroller with 8MB PSRAM. It uses a trimmed 2.8MB kernel, an EROFS+LZMA root filesystem (2.5MB), and boots from an 8MB SPI flash chip. No MMU, no SD card, no Linux board — just a soldering iron and a serial port.

Quick start: Download a release image and flash it, or test in QEMU with no hardware needed.

Flash to hardware

# Download latest release
wget https://github.com/hkcfs/mculinux/releases/latest/download/flash_r8n8.bin

# Flash (adjust port for your system)
esptool.py --chip esp32s3 --port /dev/ttyUSB0 write_flash 0x0 flash_r8n8.bin

# Connect (115200 baud)
screen /dev/ttyUSB0 115200

Test in QEMU (no hardware needed)

# Install QEMU
sudo apt install qemu-system-misc

# Boot
qemu-system-xtensa -M esp32s3 -nographic -m 8M \
  -global driver=ssi_psram,property=is_octal,value=true \
  -drive file=flash_r8n8.bin,if=mtd,format=raw

Using Web Flasher

  1. Open MCUlinux Web Flasher
  2. Select your device type
  3. Download or select firmware file
  4. Connect your ESP32-S3 via USB and hold BOOT while clicking Flash
Requirements: Chrome or Edge browser (Web Serial API). Hold the BOOT button on your ESP32-S3 when clicking Flash.

Building

MCUlinux uses Buildroot and a musl-based cross-compiler (crosstool-NG 1.25.0.183). No Docker required.

# Clone
git clone https://github.com/hkcfs/mculinux.git
cd mculinux/mculinux

# One-time setup (downloads toolchain, Buildroot, ~30 min)
make setup

# Full build (kernel + flash image + QEMU test)
make rebuild

# Or step by step
make kernel-package    # Build kernel from fork
make image DEVICE=r8n8 # Assemble flash image
make test DEVICE=r8n8  # Boot in QEMU
Note: First build takes 1-2 hours. The musl toolchain compilation is the longest step.

QEMU Testing

MCUlinux images are verified to boot in QEMU before release. You can test images yourself:

# Install QEMU for Xtensa
sudo apt install qemu-system-misc

# Boot MCUlinux
qemu-system-xtensa -M esp32s3 -nographic -m 8M \
    -global driver=ssi_psram,property=is_octal,value=true \
    -drive file=flash_r8n8.bin,if=mtd,format=raw
Note: Always run QEMU with a timeout. The image will boot to a shell prompt and wait for input. 
# With timeout (30 seconds):
timeout 30 qemu-system-xtensa -M esp32s3 -nographic -m 8M \
    -global driver=ssi_psram,property=is_octal,value=true \
    -drive file=flash_r8n8.bin,if=mtd,format=raw

Troubleshooting

Device not booting

  • Check USB connection
  • Verify baud rate (115200)
  • Try different USB port/cable

No serial output

  • Ensure correct COM port
  • Press RESET button

Web Flasher not working

  • Use Chrome or Edge (Web Serial API required)
  • Hold BOOT button while clicking Flash
  • Try ESPConnect as fallback
  • Try esptool.py as fallback
