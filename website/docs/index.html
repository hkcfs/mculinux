Getting Started

MCUlinux is a lightweight musl-based Linux distribution for ESP32-S3 microcontrollers. It runs a real Linux kernel (6.16) with a musl libc userspace, giving you a familiar shell environment on bare metal.

Note: MCUlinux currently supports ESP32-S3 only.

Supported Devices

DevicePSRAMFlashDescription
r8n88 MB8 MBBasic configuration
r8n168 MB16 MBExtended storage
r16n1616 MB16 MBHigh memory

Installation

Using esptool.py

pip install esptool

wget https://github.com/hkcfs/mculinux/releases/latest/download/r8n8.zip
unzip r8n8.zip

esptool.py --chip esp32s3 --port /dev/ttyUSB0 --baud 115200 \
    write_flash 0x0 flash_r8n8.bin

Using Web Flasher

  1. Open MCUlinux Flasher
  2. Select your device type
  3. Download or select firmware file
  4. Connect your device via USB
  5. Click Flash

Configuration

MCUlinux uses Buildroot for system configuration. The kernel and rootfs are pre-built in the release images.

# Boot sequence:
# 1. ESP32-S3 bootloader
# 2. Linux kernel (6.16)
# 3. BusyBox init
# 4. Shell on serial console

# Serial console: 115200 baud

QEMU Testing

MCUlinux images are verified to boot in QEMU before release. You can test images yourself:

# Install QEMU for Xtensa
sudo apt install qemu-system-misc

# Boot MCUlinux
qemu-system-xtensa -M esp32s3 -nographic -m 8M \
    -global driver=ssi_psram,property=is_octal,value=true \
    -drive file=flash_r8n8.bin,if=mtd,format=raw
Note: Always run QEMU with a timeout. The image will boot to a shell prompt and wait for input. 
# With timeout (30 seconds):
timeout 30 qemu-system-xtensa -M esp32s3 -nographic -m 8M \
    -global driver=ssi_psram,property=is_octal,value=true \
    -drive file=flash_r8n8.bin,if=mtd,format=raw

Build from Source

MCUlinux is built using Buildroot and a musl cross-compiler:

# Clone the repository
git clone https://github.com/hkcfs/mculinux.git
cd mculinux/mculinux

# One-time setup (downloads toolchain, Buildroot)
make setup

# Build kernel + flash image
make rebuild
Note: First build takes 1-2 hours. The musl toolchain compilation is the longest step.

Troubleshooting

Device not booting

  • Check USB connection
  • Verify baud rate (115200)
  • Try different USB port/cable

No serial output

  • Ensure correct COM port
  • Press RESET button

Web Flasher not working

  • Use Chrome or Edge (Web Serial API required)
  • Hold BOOT button while clicking Flash
  • Try esptool.py as fallback
